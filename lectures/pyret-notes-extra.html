Additional notes on using CPO (code.pyret.org)

Starting CPO

Sometimes, when you visit code.pyret.org, the page may appear to come up mostly blank. Reloading the page usually takes care of this.

Where do files go?

CPO puts all of your saved files into a directory named code.pyret.org. Unlike some other tools, CPO doesn't let you browse to a subdirectory in which to save the files. You can, however, create subdirectories and move files around manually by going into your Google Drive folder through a web browser. You can open a Pyret file that you saved in a different directory by right-clicking on it @emph{within Google Drive} and selecting "Open with", then "Pyret". The subdirectories will not show up if you go through the CPO "Open" option in the file menu.

Saving versus publishing

You will also notice that Pyret has options for both "saving" and "publishing" files. Saving stores a copy of your file in the Google Drive associated with your login account. Publishing lets you share the code file with others (like getting a share link in Google docs, if that's familiar to you). If you share a file, others can view the file, or make a personal copy, but they can't edit your copy of the file.

Important note on saving files

Pyret saves your file when you press the save button or when you run the program after having saved the file at least once. When a Pyret session/browser window has been idle for a while, the session becomes inactive and you may lose the ability to save the file until you wake up the connection by either running the program again or, in the worst case, reloading the web page. This means you should follow two guidelines:

  • Save often, especially when you plan to be away from your machine for more than a few minutes.
  • When you come back to your machine after more than a few minutes, run the program again to wake up the connection. Otherwise, changes you make to the file may get lost when you do try to save.
